## Category Filtering

Customers can filter the catalog by product categories dynamically.

### How It Works

1. **Category Bar**: Displays all available categories plus an "Todos" (All) option
2. **Active State**: Selected category shows underline and bold styling
3. **Real-time Filtering**: Products update instantly when a category is selected
4. **Category Data**: Categories are loaded from the Supabase `categories` table

```jsx
// Categories are fetched and displayed dynamically
const categories = ["Todos", "Anillos", "Collares", "Pulseras", "Aretes"]
```

<Note>
  The filtering happens client-side for instant updates, but products are loaded with their category relationships from the database.
</Note>

## Data Loading

Products are loaded from Supabase with efficient queries:

```javascript
// Main catalog query
supabase
  .from('products')
  .select('*, categories(name)')
  .order('created_at', { ascending: false })

// Single product with category
supabase
  .from('products')
  .select('*, categories(name)')
  .eq('id', productId)
  .single()

// Related products
supabase
  .from('products')
  .select('*, categories(name)')
  .eq('category_id', categoryId)
  .neq('id', currentProductId)
  .limit(4)
```

<Note>
  All product queries include the category relationship using Supabase's join syntax, ensuring category names are available without additional queries.
</Note>
