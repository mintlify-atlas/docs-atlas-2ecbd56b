## Authentication & Access Control

All admin routes are protected:

```javascript
// Check if user is authenticated and has admin role
const { data: { user } } = await supabase.auth.getUser();

if (!user) {
  redirect('/login');
}

// Check admin role
const { data: profile } = await supabase
  .from('profiles')
  .select('role')
  .eq('id', user.id)
  .single();

if (profile?.role !== 'admin') {
  redirect('/');
}
```

<Note>
  The admin panel should include role-based access control to ensure only authorized users can access administrative features.
</Note>

## Design System

The admin interface maintains the brand aesthetic while prioritizing functionality:

**Color Scheme**
- White backgrounds for content cards
- Warm cream (`#F5F1EB`, `#F9F8F6`) for subtle backgrounds
- Black for text and primary buttons
- Gold accents (`#A07F3A`, `#C4A95E`) for highlights
- Status colors: Blue (shipped), purple (processing), yellow (pending), red (alerts)

**Typography**
- Serif italic for headings ("Panel de Control")
- Uppercase tracking for labels and section titles
- Font size hierarchy: 3xl for page titles → xs for labels
- Monospace for order IDs

**Spacing & Layout**
- Generous padding (p-6, p-8)
- Consistent border radius (rounded-xl, rounded-3xl)
- Subtle borders (border-black/5)
- Soft shadows (shadow-sm, shadow-md)
