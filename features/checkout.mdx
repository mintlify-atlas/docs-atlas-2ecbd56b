The complete checkout process follows these steps:

### 1. Cart to Checkout

Customers click "Finalizar Compra" in the cart sidebar:
- Navigates to `/checkout` page
- Cart contents displayed in summary sidebar
- Empty cart redirects to empty state

### 2. Customer Information Form

Customers enter shipping details:

### 3. Server-Side Payment Setup

When form is submitted, the checkout API route processes the request:

```javascript
// POST /api/checkout/route.js

1. Verify user authentication
2. Validate cart has items
3. Fetch real prices from database
4. Create Stripe checkout session
5. Return Stripe URL to client
```

### 4. Stripe Hosted Checkout

Customer is redirected to Stripe's secure payment page:
- Stripe-hosted payment form
- PCI-compliant card processing
- Multiple payment method support
- Automatic fraud detection

### 5. Success Confirmation

After successful payment:
- Redirect to `/checkout/success?session_id={CHECKOUT_SESSION_ID}`
- Verify payment with Stripe
- Display order confirmation
- Clear cart

## Checkout API Route

The checkout API (`app/api/checkout/route.js`) handles secure payment setup.

### Authentication Check

```javascript
const supabase = await createClient();
const { data: { user } } = await supabase.auth.getUser();
if (!user) {
  return NextResponse.json({ error: "No autenticado" }, { status: 401 });
}
```

<Note>
  All checkout requests require authenticated users. Unauthenticated requests return a 401 error and redirect to login.
</Note>

### Price Verification

Critical security step - never trust client-side prices:

```javascript
// Extract product IDs from cart
const productIds = items.map((i) => i.id);

// Fetch real prices from database
const { data: products } = await supabase
  .from("products")
  .select("id, name, base_price, image_url")
  .in("id", productIds);

// Verify each item exists and use database price
const lineItems = items.map((item) => {
  const product = products.find((p) => p.id === item.id);
  if (!product) throw new Error(`Producto no encontrado: ${item.id}`);
  return {
    price_data: {
      currency: "mxn",
      unit_amount: Math.round(product.base_price * 100), // Convert to cents
      product_data: { name: product.name }
    },
    quantity: item.quantity
  };
});
```

<Tip>
  Stripe requires amounts in the smallest currency unit (centavos for MXN). The code multiplies by 100 and rounds to ensure proper conversion.
</Tip>

### Stripe Session Creation

Create a Stripe Checkout Session with complete order metadata:

```javascript
const session = await stripe.checkout.sessions.create({
  payment_method_types: ["card"],
  line_items: lineItems,
  mode: "payment",
  
  // Shipping configuration
  shipping_options: [{
    shipping_rate_data: {
      type: "fixed_amount",
      fixed_amount: { amount: shippingFee * 100, currency: "mxn" },
      display_name: "Envío estándar"
    }
  }],
  
  // Customer info
  customer_email: user.email,
  
  // Order metadata (stored with payment)
  metadata: {
    user_id: user.id,
    customer_name: customerData.fullName,
    customer_phone: customerData.phone,
    shipping_address: customerData.address,
    notes: customerData.notes || "",
    items: JSON.stringify(items) // Complete order details
  },
  
  // Redirect URLs
  success_url: `${APP_URL}/checkout/success?session_id={CHECKOUT_SESSION_ID}`,
  cancel_url: `${APP_URL}/checkout`
});
```

### Metadata Storage

All order details are stored in Stripe's session metadata:

- **User Information**: ID and email from authentication
- **Customer Details**: Name and phone for delivery
- **Shipping Address**: Complete address string
- **Order Notes**: Any special instructions
- **Items Array**: Full cart with IDs, quantities, and verified prices

<Note>
  Storing order details in Stripe metadata ensures you can retrieve complete order information even if your database is unavailable during webhook processing.
</Note>

## Shipping Configuration

Shipping fees are dynamically loaded from the `store_config` table:

```javascript
const { data: storeConfig } = await supabase
  .from("store_config")
  .select("shipping_fee")
  .single();

const shippingFee = storeConfig?.shipping_fee || 0;
```

Shipping options are only added if a fee is configured:

```javascript
shipping_options: shippingFee > 0 ? [
  {
    shipping_rate_data: {
      type: "fixed_amount",
      fixed_amount: { amount: shippingFee * 100, currency: "mxn" },
      display_name: "Envío estándar"
    }
  }
] : []
```

## Success Page

The success page (`/checkout/success/page.jsx`) confirms successful payment.

### Verification Flow

```javascript
const sessionId = searchParams.get("session_id");

// Verify session with Stripe
const res = await fetch(`/api/checkout/verify?session_id=${sessionId}`);
const data = await res.json();

if (res.ok) {
  setOrderData(data);
  clearCart(); // Remove items from cart
  setStatus("success");
}
```

## Redirect URLs

Two redirect scenarios:

**Success**: `/checkout/success?session_id={CHECKOUT_SESSION_ID}`

- Triggers order confirmation flow

- Returns to checkout form
- Cart remains intact
- User can modify information and retry
