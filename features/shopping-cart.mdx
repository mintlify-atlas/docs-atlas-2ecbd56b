## Cart Context Architecture

The cart system uses React Context to provide global state management throughout the application.

### Context Provider

Located at `context/CartContext.jsx`, the provider manages all cart operations:

```javascript
const CartContext = createContext();

export function CartProvider({ children }) {
  const [cart, setCart] = useState([]);
  const [isCartOpen, setIsCartOpen] = useState(false);
  // ... cart operations
}
```

## Persistent Storage

Cart data automatically syncs with browser localStorage.

### How It Works

**On Initial Load**
```javascript
useEffect(() => {
  const saved = localStorage.getItem('zalea_cart');
  if (saved) {
    setCart(JSON.parse(saved));
  }
  setIsLoaded(true);
}, []);
```

**On Cart Changes**
```javascript
useEffect(() => {
  if (isLoaded) {
    localStorage.setItem('zalea_cart', JSON.stringify(cart));
  }
}, [cart, isLoaded]);
```

<Note>
  The `isLoaded` flag prevents overwriting localStorage during the initial hydration, ensuring saved cart data isn't lost on page refresh.
</Note>

## Cart Item Management

### Adding Items

When a product is added to the cart:

1. **Check for Existing Item**: If the product ID already exists, increment quantity
2. **New Item**: Add product with `quantity: 1`
3. **Auto-open Sidebar**: Cart sidebar opens automatically
4. **Visual Feedback**: User sees item appear in real-time

```javascript
const addToCart = (product) => {
  setCart((prev) => {
    const existing = prev.find((item) => item.id === product.id);
    if (existing) {
      return prev.map((item) =>
        item.id === product.id
          ? { ...item, quantity: item.quantity + 1 }
          : item
      );
    }
    return [...prev, { ...product, quantity: 1 }];
  });
  setIsCartOpen(true);
};
```

### Quantity Controls

Each cart item has quantity controls:

**UI Components**
- Minus button (−): Decreases quantity by 1
- Quantity display: Current count in bold
- Plus button (+): Increases quantity by 1
- Circular border design with hover effects

**Logic**
```javascript
const updateQuantity = (id, quantity) => {
  if (quantity <= 0) {
    removeFromCart(id);
    return;
  }
  setCart((prev) =>
    prev.map((item) =>
      item.id === id ? { ...item, quantity } : item
    )
  );
};
```

<Note>
  Setting quantity to 0 or below automatically removes the item from the cart, providing a consistent experience.
</Note>

## Price Calculations

### Per-Item Display

```javascript
// Total for this item
${(item.base_price * item.quantity).toLocaleString()} MXN

// Unit price (shown when quantity > 1)
${item.base_price.toLocaleString()} c/u
```

### Cart Totals

**Subtotal**
- Sum of all items (price × quantity)
- Displayed in large serif font in footer
- Uses `toLocaleString()` for thousand separators

**Shipping**
- Calculated during checkout
- Not included in cart sidebar
- Fetched from `store_config` table during payment

<Tip>
  All prices are stored and calculated in the main currency unit (MXN), not cents. Stripe conversion to centavos happens only in the checkout API.
</Tip>

## Integration with Checkout

When users click "Finalizar Compra":

1. **Navigation**: Redirects to `/checkout` page
2. **Cart Validation**: Checkout page verifies cart is not empty
3. **Item Data**: Full cart contents passed to checkout form
4. **Price Verification**: Server verifies all prices from database
5. **Clear on Success**: Cart cleared after successful payment
