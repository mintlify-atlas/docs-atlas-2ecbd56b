## Overview

Rubik's Vintage is built with **Next.js 14+ App Router**, utilizing modern React Server Components and route groups for clean separation between public and admin interfaces.

## Route Groups

### (admin) Route Group

Protected routes requiring authentication and admin privileges. All pages under this group inherit the admin layout.

**Layout implementation** (`app/(admin)/layout.jsx`):

```jsx
import { createClient } from '@/utils/supabase/server'
import { redirect } from 'next/navigation'
import AdminShell from '@/components/admin/AdminShell'

export default async function AdminLayout({ children }) {
  const supabase = await createClient()

  const { data: { user } } = await supabase.auth.getUser()
  if (!user) redirect('/login')

  const { data: profile } = await supabase
    .from('profiles')
    .select('is_admin')
    .eq('id', user.id)
    .single()

  if (!profile?.is_admin) redirect('/')

  return <AdminShell>{children}</AdminShell>
}
```

### (public) Route Group

Public-facing routes accessible to all visitors. Includes the storefront, product pages, and authentication.

**Layout implementation** (`app/(public)/layout.jsx`):

```jsx
import Navbar from "@/components/public/Navbar";
import AnnouncementBar from "@/components/public/AnnouncementBar";

export default function PublicLayout({ children }) {
  return (
    <div className="flex flex-col min-h-screen">

      <footer className="border-t border-gray-100 py-12 bg-[#F9F8F6]/80">
        {/* Footer content */}
      </footer>
    </div>
  );
}
```

## Key Directories

### `/app`

Next.js App Router directory containing:
- **Route groups** for layout separation
- **API routes** in `/api`
- **Server Components** by default
- **Middleware** for auth protection

### `/components`

Reusable React components organized by domain:
- **`admin/`** - Admin panel UI components
- **`public/`** - Storefront components

All components use **`"use client"`** directive when needed for interactivity.

### `/context`

React Context providers for global state:
- **`CartContext.jsx`** - Shopping cart state with localStorage persistence

### `/utils/supabase`

Supabase client utilities:
- **`client.js`** - Browser client for client components
- **`server.js`** - Server client for server components and API routes
- **`admin.js`** - Admin client with elevated privileges

## Component Organization

### Client vs Server Components

**Server Components (default):**
- Page layouts
- Static content
- Data fetching from Supabase

**Client Components (`"use client"`):**
- Interactive UI (buttons, forms)
- State management (cart, modals)
- Browser APIs (localStorage)

### Example: ProductCard Component

Location: `components/public/ProductCard.jsx`

```jsx
"use client";
import Image from "next/image";
import { useState } from "react";
import QuickViewModal from "./QuickViewModal";

export default function ProductCard({ product }) {
  const [showModal, setShowModal] = useState(false);

  return (
    <>
      <div className="group cursor-pointer" onClick={() => setShowModal(true)}>
        <div className="relative aspect-3/4 bg-white overflow-hidden rounded-xl mb-4 shadow-sm">
          <Image
            src={product.image_url}
            alt={product.name}
            fill
            sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
            className="object-cover transition-transform duration-700 group-hover:scale-110"
          />
          {/* Quick view overlay */}
        </div>
        <h3 className="text-[11px] uppercase tracking-widest">{product.name}</h3>
        <p className="text-sm font-serif italic text-[#A07F3A]">
          ${product.base_price?.toLocaleString()} MXN
        </p>
      </div>
      {showModal && (
        <QuickViewModal
          product={product}
          onClose={() => setShowModal(false)}
        />
      )}
    </>
  );
}
```
