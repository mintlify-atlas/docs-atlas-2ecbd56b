Get your Rubik's Vintage e-commerce platform up and running on your local machine in just a few minutes.

## Prerequisites

Before you begin, make sure you have:

- **Node.js 20+** installed on your machine
- **npm** or **yarn** package manager
- **Git** for cloning the repository
- A **Supabase account** (free tier works great)
- A **Stripe account** (test mode is fine for development)

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Rubik's Vintage repository to your local machine:

    ```bash
    git clone https://github.com/GonzDv/Rubiks-Vintage.git
    cd Rubiks-Vintage
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using npm:

    <CodeGroup>
    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>

    This will install:
    - Next.js 16.1.6
    - React 19.2.3
    - Supabase client libraries
    - Stripe SDK
    - Tailwind CSS 4
    - TypeScript 5
  </Step>

  <Step title="Set up environment variables">
    Create a `.env.local` file in the root directory with your credentials:

    ```bash .env.local
    # Supabase Configuration
    NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
    SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

    # Stripe Configuration
    STRIPE_SECRET_KEY=your_stripe_secret_key
    NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key

    # Application URL
    NEXT_PUBLIC_APP_URL=http://localhost:3000
    ```

    <Note>
      Don't have these credentials yet? Check the [Setup Guide](/setup) for detailed instructions on obtaining them.
    </Note>
  </Step>

  <Step title="Run the development server">
    Start the Next.js development server:

    ```bash
    npm run dev
    ```

    The application will be available at **http://localhost:3000**
  </Step>

  <Step title="Explore the platform">
    Open your browser and navigate to:

    - **http://localhost:3000** - Public storefront
    - **http://localhost:3000/login** - Login page
    - **http://localhost:3000/dashboard** - Admin panel (after login)

    <Tip>
      The admin panel is protected. You'll need to create a user account in Supabase first. See the [Setup Guide](/setup) for details.
    </Tip>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Environment variables not loading">
    Make sure your `.env.local` file is in the root directory (same level as `package.json`). Restart the dev server after creating or modifying environment variables.
  </Accordion>

  <Accordion title="Supabase connection errors">
    Verify your `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are correct. You can find these in your Supabase project settings under **API**.
  </Accordion>

  <Accordion title="Stripe errors during checkout">
    Ensure you're using your Stripe **test mode** keys during development. Test keys start with `pk_test_` and `sk_test_`.
  </Accordion>

  <Accordion title="Database errors">
    The database tables need to be created in Supabase. Follow the [Database Setup](/deployment/database-setup) guide to create the required schema.
  </Accordion>
</AccordionGroup>

<Warning>
  Never commit your `.env.local` file to version control. It's already included in `.gitignore` to prevent accidental exposure of credentials.
</Warning>
